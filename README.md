# Індивідуальне завдання. Мікросервісна система REST

Цей репозиторій містить навчальний макет мікросервісної системи REST з навмисними багами для вправи з пошуку та виправлення помилок.  
Сервіси: **auth-service**, **product-service**, **order-service**.

---

## Технології
- **FastAPI** — для реалізації REST API  
- **Uvicorn** — ASGI сервер для запуску сервісів  
- **Docker** і **Docker Compose** — для контейнеризації та запуску усіх сервісів одночасно  

---

## Важливі принципи
- **Безпека:** не логуйте паролі та токени; не передавайте їх у URL.  
- **HTTP статуси:** використовуйте правильні коди статусу та явні повідомлення про помилки для надійної інтеграції.  
- **Схема даних:** узгоджена схема зменшує ризик помилок інтеграції.  
- **Pydantic:** мінімум — використати Pydantic-схеми; максимум — повний контракт та контрактні тести.

## Мета роботи

Ознайомитися з:
- Взаємодією між мікросервісами;
- Розумінням REST-контрактів;
- Типовими помилками інтеграції.

---

## Завдання

1. Створити репозиторій на GitHub із початковим кодом.
2. Знайти помилки.
3. Створити модульні тести (`pytest`), які покажуть наявність помилок. Один тест має перевіряти одну помилку.
4. Проаналізувати важливість знайдених помилок.
5. Виправити помилки. Кожне виправлення оформити в окрему гілку git та створити Pull Request.
6. За допомогою тестів показати, що помилки виправлено.
